Phase 2 — Hooks de données (Jours 2-4)
Un fichier par entité dans hooks/queries/. Chaque fichier contient :

useXxx() → lecture (useQuery)
useCreateXxx() → création (useMutation)
useUpdateXxx() → modification
useDeleteXxx() → suppression
Exemple pour les chantiers (hooks/queries/useChantiers.ts) :

export function useChantiers() {
return useQuery({
queryKey: keys.chantiers,
queryFn: () => supabase.from('chantiers').select('*').order('created_at', { ascending: false }),
select: (data) => data.data?.map(toChantier) ?? [],
});
}
export function useCreateChantier() {
const queryClient = useQueryClient();
return useMutation({
mutationFn: (chantier) => supabase.from('chantiers').insert(chantier),
onSuccess: () => queryClient.invalidateQueries({ queryKey: keys.chantiers }),
});
}

Phase 3 — Realtime Supabase (Jour 5)
Créer hooks/useRealtimeSync.ts — un seul hook qui gère toutes les subscriptions :

export function useRealtimeSync(chantierId: string | null) {
const queryClient = useQueryClient();
useEffect(() => {
const channel = supabase.channel('realtime-sync')
.on('postgres_changes', { event: '', schema: 'public', table: 'reserves' }, () => {
queryClient.invalidateQueries({ queryKey: keys.reserves(chantierId ?? undefined) });
})
.on('postgres_changes', { event: '', schema: 'public', table: 'chantiers' }, () => {
queryClient.invalidateQueries({ queryKey: keys.chantiers });
})
// ... autres tables
.subscribe();
return () => { supabase.removeChannel(channel); };
}, [chantierId, queryClient]);
}
Avantage : quand Supabase pousse un changement, React Query re-fetch automatiquement et tous les composants abonnés se mettent à jour.

Phase 4 — Micro-contexte UI (Jour 5)
Remplacer le gros AppContext par un contexte minimaliste uniquement pour l'état UI (ce qui n'est pas du server-state) :

interface AppUIState {
activeChantierId: string | null;
setActiveChantierId: (id: string | null) => void;
activeChantier: Chantier | null; // dérivé
lastReadByChannel: Record<string, string>;
setChannelRead: (channelId: string, ts: string) => void;
}
L'activeChantierId est persisté dans AsyncStorage avec un simple useEffect — plus besoin du reducer complexe.

Le bug de l'auto-sélection disparaît totalement : useChantiers() retourne son propre état loading/data, et activeChantierId se fixe dans un useEffect qui réagit quand data arrive :

const { data: chantiers } = useChantiers();
useEffect(() => {
if (!activeChantierId && chantiers?.length) {
setActiveChantierId(chantiers[0].id);
}
}, [chantiers, activeChantierId]);
