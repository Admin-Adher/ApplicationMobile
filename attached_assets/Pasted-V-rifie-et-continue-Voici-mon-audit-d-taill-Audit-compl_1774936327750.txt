⚠️ Lacunes et incohérences identifiées
1. Gestion des entreprises dupliquée (incohérence majeure)
L'onglet Équipes et l'onglet Admin permettent tous les deux de créer/modifier/supprimer des entreprises avec exactement les mêmes formulaires. C'est redondant et risque de créer de la confusion sur qui fait quoi.

2. Plans interactifs : un seul niveau par bâtiment
On ne peut afficher qu'un plan par bâtiment, sans distinction de niveaux (RDC, R+1, R+2…). Pour un chantier réel, c'est une lacune importante — une réserve "Niveau 3" ne peut pas être localisée sur le bon plan.

3. Aucun lien réserve ↔ tâche corrective
Quand on crée une réserve, il n'y a aucune possibilité de lui associer automatiquement une tâche corrective et vice versa. Ces deux modules sont totalement déconnectés alors qu'ils devraient être liés : une réserve ouverte devrait pouvoir générer une tâche, et cette tâche devrait apparaître dans le détail de la réserve.

4. Pas d'historique journalier du personnel
La présence (présents/heures) est une valeur instantanée, écrasée à chaque mise à jour. Il n'existe aucun historique de pointage par date, ce qui rend impossible de retrouver les effectifs d'une journée passée dans les rapports.

5. Le rapport PDF ne couvre pas les réserves en détail
Le rapport journalier liste uniquement le personnel et les tâches en cours. Les réserves ouvertes, critiques ou en retard n'y apparaissent pas sous forme de liste. Le rapport hebdomadaire liste seulement les critiques. Pour un vrai rapport BTP, la liste exhaustive des réserves avec leur statut est indispensable.

6. Date picker en texte libre — source d'erreurs
Toutes les dates (échéances réserves, tâches) sont saisies manuellement au format JJ/MM/AAAA dans un champ texte. Un sélecteur de date natif (DateTimePicker) serait bien plus fiable et standard.

7. L'onglet Admin visible par tous les rôles
L'écran Admin (onglet dans la barre de navigation) s'affiche dans les tabs pour tous les utilisateurs. Il n'y a aucun guard dans le tab layout pour le cacher aux non-admins — seulement une validation côté Supabase, mais l'onglet reste visible.

8. Pas de notifications push
Aucune alerte temps réel : nouvelle réserve critique, tâche passant en retard, message non lu. Pour une application de chantier, c'est un manque fonctionnel significatif.

9. Messagerie non synchronisée en temps réel via Supabase
Les messages sont stockés localement (AsyncStorage). Il n'y a pas de connexion à Supabase Realtime pour la messagerie — ce qui signifie que deux utilisateurs sur deux appareils ne verront pas les mêmes messages en temps réel.

10. Pas de module Sécurité/Incidents
Il n'existe aucun écran pour déclarer un incident ou accident de sécurité sur le chantier. C'est une obligation légale dans le BTP (document unique, registre de sécurité). Cette fonctionnalité est absente.

11. Pas de suivi météo dans les rapports
Le rapport journalier ne mentionne pas la météo, alors que c'est un élément standard du rapport de chantier (conditions météo influençant l'avancement et la présence).

12. Nom de projet "Projet Horizon" figé dans le code
Le nom du projet est hardcodé dans le dashboard et dans les rapports. Il n'y a aucune page de configuration pour changer le nom du projet, ni de gestion multi-projets.

13. Photos module déconnecté des réserves
L'écran Photos permet de prendre/importer des photos indépendamment, mais ces photos n'ont aucun lien avec une réserve spécifique. Les photos de réserves et les photos de chantier générales vivent dans des silos séparés.

14. Les tâches n'ont ni photo ni pièce jointe
Contrairement aux réserves, une tâche ne peut pas avoir de photo ou de document joint. Pourtant, pour une tâche corrective BTP, il est courant de joindre une photo "avant/après".

15. Pas de recherche globale
Il n'existe pas de recherche cross-modules. Pour retrouver quelque chose, il faut savoir dans quel module chercher (réserves, tâches, documents ou messages).
